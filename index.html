Key Technical Decisions

Why AWS Kiro?

Native AWS agent framework with built-in Bedrock integration, IAM controls, and audit trails — no third-party LLM orchestration risk.

Why Amazon Bedrock?

Keeps all LLM inference inside the AWS security boundary. No data leaves the VPC. Meets enterprise compliance requirements.

Why Python self-healing?

Bedrock reasons about findings; Python executes deterministic remediations via SSM Run Command — separating AI judgement from action execution.

GitLab Pipeline + Qualys Automation

Infor Inc · 2022–2023

Security Compliance Pipeline · GitLab CI/CD + Qualys

80% compliance ↑
Problem

Vulnerability scanning was disconnected from the deployment pipeline — security findings sat in Qualys dashboards but had no automated path to remediation, leaving environments exposed for weeks.

Approach

Built GitLab CI/CD pipeline stages that called the Qualys API at deploy time, parsed severity findings with Python, auto-remediated known CVEs, and blocked deployments on critical unresolved issues.

Result

Environmental security compliance improved by 80%. Vulnerability dwell time dropped from weeks to hours. Security became a first-class step in every deployment.

Key Technical Decisions

Pipeline-native scanning

Integrating Qualys directly into GitLab stages (not as a separate process) ensured every deploy was scanned — no exceptions, no manual triggers.

Severity-tiered response

Python triage logic separated Critical/High (block + alert) from Medium/Low (auto-remediate + log), preventing alert fatigue while ensuring zero critical bypass.
